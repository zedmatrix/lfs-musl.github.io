7.14. CMake-4.2.3

7.14.1. Introduction to CMake

The CMake package contains a modern toolset used for generating Makefiles. It is a successor of the auto-generated configure script and aims to be platform- and compiler-independent. A significant user of CMake is KDE since version 4.

Approximate build time: 2.2 SBU (add 4.3 SBU for tests, both using parallelism=4)
Required disk space: 610 MB (add 1.6 GB for tests)

7.14.2. Installation of CMake

Prepare CMake by running the following commands: 

sed -i '/"lib64"/s/64//' Modules/GNUInstallDirs.cmake

./bootstrap --prefix=/usr        \
            --system-libs        \
            --mandir=/share/man  \
            --no-system-libs     \
            --docdir=/share/doc/cmake-4.2.3
            -- -DCMAKE_USE_OPENSSL=OFF

Now Compile CMake by running the following commands: 

make

Now, Install the package, issue: 

make install

7.14.3. Command Explanations

sed ... Modules/GNUInstallDirs.cmake: This command disables applications using cmake from attempting to install files in /usr/lib64/.

--no-system-libs: This switch tells the build system to install all needed libraries.

-DCMAKE_USE_OPENSSL=OFF: This switch disables the use of OpenSSL as its not installed yet and not needed in this stage.

7.14.4. Contents

Installed Programs: ccmake, cmake, cmake-gui (optional), cpack, and ctest
Installed Libraries: None
Installed Directories: /usr/share/cmake-4.2 and /usr/share/doc/cmake-4.2.3

Short Descriptions

ccmake

is a curses based interactive frontend to cmake

cmake

is the makefile generator

cpack

is the CMake packaging program

ctest

is a testing utility for cmake-generated build trees
