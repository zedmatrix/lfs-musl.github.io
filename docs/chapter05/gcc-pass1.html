GCC requires the GMP, MPFR and MPC packages. As these packages may not be included in your host distribution, they will be built with GCC. Unpack each package into the GCC source directory and rename the resulting directories so the GCC build procedures will automatically use them: 
tar -xf ${YSRC}/gmp-6.3.0.tar.xz && mv -v gmp-6.3.0 gmp &&
tar -xf ${YSRC}/mpc-1.3.1.tar.gz && mv -v mpc-1.3.1 mpc &&
tar -xf ${YSRC}/mpfr-4.2.2.tar.xz && mv -v mpfr-4.2.2 mpfr &&
sed -e '/m64=/s/lib64/lib/' -i.orig gcc/config/i386/t-linux64
mkdir -v Ybuild && cd Ybuild &&
../configure --target=$LFS_TGT --prefix=$LFS/tools --with-glibc-version=2.42 \
--with-sysroot=$LFS --with-newlib --without-headers --enable-default-pie \
--enable-default-ssp --disable-nls --disable-shared --disable-multilib \
--disable-threads --disable-libatomic --disable-libgomp --disable-libquadmath \
--disable-libssp --disable-libvtv --disable-libstdcxx --enable-languages=c,c++ &&
make
Create a full version of the internal header using a command that is identical to what the GCC build system does in normal circumstances:
cd Ybuild && make install &&
cd .. &&
cat gcc/limitx.h gcc/glimits.h gcc/limity.h > \
`dirname $($LFS_TGT-gcc -print-libgcc-file-name)`/include/limits.h
